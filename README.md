# RL для дообучения Vision-Language-Action моделей

Прототип к выпускной квалификационной работе **«Исследование применения
обучения с подкреплением для повышения эффективности управления роботами
с использованием визуально-языковых моделей»** (Университет ИТМО, ФСУиР,
направление 15.03.06 «Мехатроника и робототехника», 2026).

Репозиторий содержит код двух треков интеграции RL и VLA/VLM:

1. **Основной трек** — residual RL-дообучение готовых VLA-политик на бенчмарке
   LIBERO: базовая VLA заморожена, обучается лёгкий SAC-агент, выдающий
   остаточную поправку к действию.
2. **Дополнительный трек** — автоматическая генерация функции вознаграждения
   визуально-языковой моделью (Qwen2.5-VL-7B) с итеративной рефлексией, среда
   MetaWorld (сценарий без предобученной VLA).

## Ключевая идея

Итоговое действие формируется как сумма «замороженного» предсказания VLA и
обучаемой остаточной поправки:

```
a_t = π_VLA(o_t, l) + Δπ_RL(s_t)
```

где `π_VLA` — замороженная Vision-Language-Action политика, `Δπ_RL` —
двухслойный (256/256, ReLU) SAC-агент, а `s_t` — конкатенация визуального
эмбеддинга (512), проприоцепции (9) и текстового эмбеддинга инструкции (768).
Вознаграждение стадийное: `Reach → Grasp → Transport → Place`.

## Основные результаты (LIBERO, средний SR по 4 наборам)

| Модель          | До RL  | После RL | Δ        |
|-----------------|:------:|:--------:|:--------:|
| OpenVLA-OFT-7B  | 97,1   | **98,6** | +1,5     |
| UniVLA-7B       | 95,0   | 96,8     | +1,8     |
| π₀-3B           | 94,0   | 96,1     | +2,1     |
| CogACT-8B       | 93,0   | 95,4     | +2,4     |
| SmolVLA-2.2B    | 92,5   | 95,3     | +2,8     |
| π₀-fast-3B      | 90,6   | 93,8     | +3,2     |
| SpatialVLA-4B   | 88,0   | 91,5     | +3,5     |
| Octo-Small      | 86,0   | 89,9     | +3,9     |
| RT-1-X          | 82,9   | 87,0     | +4,1     |

Residual SAC со стадийной наградой устойчиво повышает SR всех девяти моделей
(+1,5…+4,1 п.п.). Полная разморозка VLA (direct SAC) приводит к деградации
(−0,6 п.п.) из-за катастрофического забывания, поэтому заморозка базовой
политики предпочтительна.

> Числовые значения иллюстрируют выводы работы; для воспроизведения нужны
> веса соответствующих VLA-моделей и установленный бенчмарк LIBERO.

## Структура репозитория

```
bachelor-thesis/
  configs/                 # Hydra-конфиги экспериментальных серий
    series1_baseline.yaml
    series2_rl_methods.yaml
    series3_residual_all.yaml
    series4_ablation.yaml
    series5_vlm_reward.yaml
  src/vla_rl/
    evaluate_vla.py        # инференс 9 VLA-политик на LIBERO (серия 1)
    residual_rl_train.py   # residual RL-дообучение (SAC/PPO, серии 2–4)
    stage_reward.py        # стадийная функция вознаграждения (4 фазы)
    vlm_reward_gen.py      # VLM-кодогенерация награды (Qwen2.5-VL, серия 5)
  scripts/
    run_series1.sh ... run_series5.sh
  results/
    libero/                # CSV с SR, чекпоинты, кривые обучения
    metaworld/reward_code/ # сгенерированные функции награды
```

## Установка

```bash
python -m venv .venv && source .venv/bin/activate   # Python 3.12
pip install -r requirements.txt
```

Для основного трека дополнительно требуются установленные LIBERO/ROBOSUITE и
веса VLA-моделей; для дополнительного трека — MetaWorld и доступ к Qwen2.5-VL
(через vLLM). Эти зависимости не ставятся автоматически — см. ссылки в
`requirements.txt`.

## Запуск

```bash
# Серия 1: baseline девяти VLA на LIBERO
bash scripts/run_series1.sh

# Серия 2: сравнение схем RL на OpenVLA-OFT
bash scripts/run_series2.sh

# Серия 3: residual SAC + stage для всех 9 VLA
bash scripts/run_series3.sh

# Серия 4: абляция компонентов
bash scripts/run_series4.sh

# Серия 5: VLM-кодогенерация награды на MetaWorld
bash scripts/run_series5.sh
```

Конфигурации задаются через [Hydra](https://hydra.cc/); параметры можно
переопределять из командной строки, например:

```bash
python -m vla_rl.residual_rl_train --config-name series3_residual_all \
    model=openvla_oft suite=libero_long seed=42 total_steps=300000
```

## Вычислительная среда

NVIDIA A100 (40 ГБ), 32 CPU, 128 ГБ RAM. Полный цикл серии 3
(9 моделей × 4 набора LIBERO) — около 200 GPU-часов. SAC берётся из
Stable-Baselines3, буфер `1e6`, бюджет 300 000 шагов на набор, 3 seed (7, 13, 42).
